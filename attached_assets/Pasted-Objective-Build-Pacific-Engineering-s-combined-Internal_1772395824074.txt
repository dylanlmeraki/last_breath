Objective
Build Pacific Engineering's combined Internal + Client Portal as a monorepo (Option C). This Repl centralizes around the Internal Portal (40+ pages) while fully integrating Client Portal structures (18+ pages). Remove all Base44 dependencies, implement session-based auth (Lucia-pattern with Express-session), build the full Drizzle schema (26 tables), wire up the generic entity CRUD factory, configure react-router-dom with subdomain detection, and migrate the InternalLayout + core pages — all with no text content changes, UI/UX-only upgrades, and full responsiveness.

Key constraints:

Do NOT change any text content — UI/UX upgrades only
Remove all Base44 dependencies (@base44/sdk, base44Client.js, VisualEditAgent, NavigationTracker, appParams)
Use react-router-dom (NOT wouter) for routing
Session-based auth with Argon2 password hashing (Lucia-pattern, Express-native)
Full responsiveness across both portals
TypeScript where reasonable
